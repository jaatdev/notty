## üì± Mobile Considerations

On mobile devices and tablets:
- Keyboard shortcuts are disabled or limited
- Touch gestures replace keyboard shortcuts
- Swipe left/right for flashcard navigation
- Tap to flip flashcards
- Pull-to-refresh on lists

## üîß Accessibility

All interactive elements support:
- Tab navigation
- Enter/Space activation
- Screen reader announcements
- Focus indicators

## üí° Tips

1. **Learn gradually**: Start with `Ctrl+N` (new note) and `Ctrl+S` (save)
2. **Modal navigation**: `Escape` always closes the current modal
3. **Editor modes**: Use `Ctrl+Shift+M` to switch between WYSIWYG and Markdown
4. **Quick search**: `Ctrl+K` is your best friend for fast navigation
5. **Preview before saving**: Use `Ctrl+P` to see formatted output

## üêõ Troubleshooting

**Shortcut not working?**
- Check if focus is in an input field (some shortcuts disabled)
- Ensure no browser extension conflicts
- Try alternative shortcuts when available
- Check browser's default shortcuts (Ctrl+N opens new tab in some browsers)

**Conflicts with browser shortcuts:**
- `Ctrl+N` may open new tab: Use `Ctrl+Shift+N` instead
- `Ctrl+P` may print: Modal shortcuts override when modal is open
- `Ctrl+S` may save page: Application prevents default behavior
